Unlike random forests, neural networks are sensitive to the scale of the input data. Consider training a model that has variables Household Income and Number of Children. The income variable has values likely in the 10,000s, whereas number of children may range from 0-5. For many algorithms, large values can dominate small values, so a common practice is to rescale each variable to be on a common scale. Some popular rescalings for neural networks are:

  • Scale each column to range from 0 to 1
  • Scale each column to range from -1 to 1
  • Scale each column such that the standard deviation is 1, and center the data so the mean is 0 (z-scoring or whitening)

If youre interested, theres good reading about rescaling here.

Below is code that rescales each input to range from 0 to 1. The apply() function is useful for applying a function to each column of a dataframe (here, we do min and max). And the scale() function will take care of rescaling. Run the chunk below to rescale each input column to 0-1.

mins = apply(iris[,1:4], 2, min) # Get the minimum value of each column
maxs = apply(iris[,1:4], 2, max) # Get the maximum value of each column

# Apply the re-scaling to the first four columns
iris[,1:4] = scale(iris[,1:4], center = mins, scale = maxs - mins)

Ideally, a fair model will not base predictions on variables we believe are irrelevant to decision-making. What constitutes irrelevant will vary by the situation, which is why domain expertise is needed in any data science endeavor. In justice settings, we ought to strive for fairness by constructing models that are unbiased to qualities such as race or gender. We saw last time that this is complicated!

In this particular dataset, Gender was coded as F and M, and all rows with F contained missing data in some columns. We might imagine women are given a different assessment than are men (if someone knows the answer, let me know!). This is a measurement bias we unfortunately have no control over. For simplicity, the Gender column has been removed. Race is coded either White or Black, which I converted to 0 or 1 when cleaning. To ensure Race is treated by the model without bias, lets check that the probabilities of getting a true outcome are roughly the same for each Race. Last time we saw this called demographic/statistical parity.

# Check for "statistical parity" - get probability of outcome for each level of Race
r0_prob = sum(recid$Race == 0 & recid$Recidivism_Within_3years == 'true') / sum(recid$Race == 0)
r1_prob = sum(recid$Race == 1 & recid$Recidivism_Within_3years == 'true') / sum(recid$Race == 1)

# Parity is difference in probability of outcome between two groups
parity = abs(r0_prob - r1_prob)
parity * 100 # Express as a percentage
